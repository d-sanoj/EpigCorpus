# Data

This folder contains the EDCS inscription data exports generated by the scrapers.

## Files

### Incremental scraper outputs (`src/edcs_scraper.py`)

- `edcs_inscriptions.jsonl` — Main export in JSON Lines format (one inscription record per line). This is append-safe for incremental updates and resumable scraping.
- `edcs_inscriptions.tsv` — Same records exported as tab-separated values using a fixed column schema.
- `edcs_checkpoint.json` — Temporary resume marker written during a scrape run. Deleted automatically after a full successful run. Not tracked in git.

#### Record-level format details

- `edcs_inscriptions.jsonl`:
  - Each line is an independent JSON object.
  - `category` is stored as a JSON list (e.g. `["tituli sacri", "viri"]`).
  - Recommended read pattern: `pd.read_json("data/edcs_inscriptions.jsonl", lines=True)`.
- `edcs_inscriptions.tsv`:
  - Tab-delimited flat export of the same records.
  - `category` is flattened as a pipe-separated string (`value1 | value2 | ...`).
  - Best for spreadsheets and SQL-style imports.

### Lookup / reference data

- `edcs_lookup.json` — Multilingual lookup table for controlled-vocabulary fields used in the inscription records. Contains two sections:
  - `material` (21 entries) — Maps Latin material codes (e.g. `aes`, `lapis`, `opus figlinae`) to their translations in English, German, French, Italian, and Spanish.
  - `gattung` (37 entries) — Maps Latin inscription-type codes (e.g. `tituli sepulcrales`, `tituli sacri`) to their translations in the same five languages.

  Use this file to decode the `material` and `category` fields in the main inscription exports.

## Generation

Run the incremental scraper from the project root to create or update the inscription exports:

```bash
python src/edcs_scraper.py
```

Each run checks the EDCS API for new records and appends only those not already present locally.

## Notes

- Data files are generated on-demand and excluded from git (see `.gitignore`).
- The checkpoint file enables safe resume after interruptions.
- `edcs_lookup.json` is a reference file and only updates when new lookup information is available.
